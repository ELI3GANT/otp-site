OTP Terminal Knowledge Brain
============================

Purpose
-------
Private retrieval layer for OTP Terminal to support lead triage, package guidance, document routing, and manual-review-first operations.

Scope
-----
- Core business knowledge is indexed from uploaded PDF/DOCX files.
- Per-client and per-lead records stay in operational tables (`leads`, `contacts`).
- Recommendations are generated by searching indexed knowledge chunks, then applying OTP-specific business rules.

Current Pipeline
----------------
1. Admin uploads PDF/DOCX in OTP Terminal Knowledge Brain panel.
2. Server extracts text, chunks content, computes vectors, stores index records.
3. Lead recommendation endpoint retrieves top-matching chunks by cosine similarity.
4. Rule engine returns structured JSON for admin review:
   - `lead_summary`
   - `recommended_package`
   - `quote_range`
   - `required_documents`
   - `next_action`
   - `admin_notes`
   - `draft_client_reply`
5. Nothing is sent automatically; admin approval is required before outgoing communication.

Business Safety Defaults
------------------------
- Always require agreement + invoice/deposit workflow before kickoff.
- Default deposit requirement: 50 percent.
- Add NDA for confidential/unreleased/private/sensitive projects.
- Add adult media release for identifiable adult likeness/voice/performance.
- Use pricing guide/package structure as source of truth.
- Favor `The Engine` when scope is growth-stage but not clearly micro or enterprise.
- Flag unclear scope for manual review.
